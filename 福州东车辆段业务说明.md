# 福州东车辆段业务说明

## 1. 文档定位

### 1.1 目的

本文档用于定义福州东车辆段的业务事实、场区主数据、作业能力和标准业务流程，目标是让不熟悉现场的人也能据此建立稳定的领域模型和主数据表，而不是继续从旧文件和旧代码里反推业务。

### 1.2 与《福州东调车业务说明》的分工

1. 本文档只保留车辆段业务事实、场区口径、线路能力、工位能力和典型业务流程。
2. 调车输入合同、目标表达、合法性校验、排位规则、勾计划输出和异常返回，统一放在《福州东调车业务说明》。
3. 两份文档出现理解冲突时，先以本文的业务事实为准，再由《福州东调车业务说明》定义调车实现要求。

## 2. 业务背景与目标

### 2.1 车辆段作业对象

车辆段是铁路货车的检修场所，车辆会在段内完成预修、调梁、洗罐、抛丸、油漆、大库检修、卸轮装卸等不同业务。
调车作业的核心任务，是使用调车机把车辆从当前股道送到目标股道、目标作业区或目标台位，并在工序完成后继续送往下一目标，直到全部工序结束并出段。

### 2.2 当前业务痛点

1. 人工编制钩计划耗时长，依赖经验，计划质量波动大。
2. 现场存在错办、挤岔、撞档、多拉少挂等安全风险。
3. 连结员需要频繁行走、摘挂、扳岔和安撤防护装置，作业强度高。
4. 人、车、线、设备、计划之间信息割裂，协同效率低。

### 2.3 建设目标

1. 安全目标：减少人工错办、挤岔、撞档、多拉少挂和防护装置遗漏风险。
2. 效率目标：缩短钩计划编制时间，降低人工强度，提升人车线协同效率。

### 2.4 相关外部系统

需求说明提到，后续业务会依赖以下系统提供输入或执行支撑：

1. 全息数字孪生作业管理平台：展示线路、车辆、设备和计划状态。
2. 车辆运行态势感知系统：提供车号、位置、车型、空重、修程等状态数据。
3. 调车安全防护系统：用于路径安全、防护装置和现场风险校验。
4. 电动转辙机远程集中控制系统：用于路径执行和道岔控制。
5. 调车机智能辅助驾驶系统：用于路径导航、计划下发和执行辅助。


## 3. 术语与对象模型

### 3.1 术语

| 术语 | 定义 |
| --- | --- |
| 股道 | 可走行、停放或作业的具体线路 |
| 区域 | 若干股道组成的业务分区，如预修区、功能作业区、运营能力区 |
| 作业区 | 某条股道上真正执行设备作业或台位作业的那一段范围 |
| 台位 | 大库、洗罐、油漆、调棚等作业区内的具体停放位 |
| 临停 | 仅允许调车过程中短时甩车，不允许长期停留 |
| 缓存 | 为后续取送、清线、折返服务的中间停放用途，不是独立场区 |
| 缓冲区 | 辆渡区及其横移过渡区域，只承担横移和过渡，不作为普通存车区 |
| 对位 | 把车送入特定作业区或特定台位 |
| 非对位业务 | 只要求到达指定线或指定能力区，不要求精确落到固定台位 |
| 可停车有效长度 | 股道用于停车、留车、容量校验的有效长度，不含警冲标和岔区不可停车部分 |
| 实际物理距离 | 车辆运行或倒车时用于判断“走不走得过去、会不会干涉”的分支物理距离，按同一分支下各片段长度求和 |
| 关门车 | 不具备接风管/自动制动条件的车辆属性，不是独立车型 |

### 3.2 基本关系

1. 一条股道可属于一个区域。
2. 一条股道可包含零个或多个作业区。
3. 一个作业区可包含零个或多个台位。
4. 一辆车任一时刻只能位于一条股道上的一个顺位或一个台位。
5. 当前接口中的车辆属性字段一次只表达一个离散属性值，允许为空，不做多属性并存建模。

## 4. 场区结构与股道主数据

### 4.1 场区业务分区

福州东车辆段按业务可分为八类区域：

1. 进段接车区：承接车站送入段内的车辆，作为待分流、待判定和后续形成本轮目标前的接车区域，主要对应 `存5北`、`存5南`。
2. 一般存车区：承担普通停放、缓存和进出作业区前后的过渡停放，主要包括 `存1`、`存2`、`存3`、`机北`、`调北`、`洗北` 以及 `修1库外` 至 `修4库外`。
3. 出段集结区：承担修竣车辆正式集结和交站，正式出段集结线为 `存4北`；`存4南` 不属于正式出段集结区，只作过程临停。
4. 预修区：承担预修能力，主能力线为 `预修`、`机棚`；调棚南端无编号尽头预修段也归入本区，但调棚整条线路不整体归入预修区。
5. 功能作业区：承担调梁、洗罐、抛丸、油漆和大库检修等功能作业，主要包括调棚 `1` 至 `4` 号位、洗南 `1` 至 `3` 号位、`抛`、`油`、`修1库内` 至 `修4库内`；现场不再把段修区、厂修区写成两个固定物理区域。
6. 临停区：只用于调车过程中的临时甩车和阶段性过渡停放，不作为正式终点，主要包括 `临1` 至 `临4` 和 `存4南`。
7. 运营能力区：承担卸轮、装卸、托运等运营作业，当前由 `轮` 单独构成，不并入功能作业区。
8. 联线与辆渡缓冲区：承担线路联通、路径通过和横移过渡，主要包括 `联6`、`联7`、`渡1` 至 `渡13` 及相关缓冲区域，不作为普通存车区。

补充口径：

1. 调棚属于复合线路：`1` 至 `4` 号位是功能作业区内的调梁对位工位，最南尽头是预修区内的无编号预修段。
2. 机库不单独列入上述八类区域，按专用线管理；线内包含 `1` 个称重功能位，机库主线既可在调车过程中配合称重，也可作为最终存车线和最终目的地。

### 4.4 运行物理距离口径

股道主数据总表记录的是运行过程的分支物理距离，不是股道可停车有效长度。相关数据在本文只用于运行、倒车和干涉判断。

1. 该表只用于判断车辆运行过程中是否存在干涉、倒车距离是否放得下、路径物理距离是否足够以及经过路径判断。
2. 表中第 2 列 `线路` 只按道岔分支建键；空单元格表示沿用上一条非空 `线路`。
3. 表中第 3 列 `线路组成` 是该分支的组成片段，第 4 列 `长度` 是片段长度。
4. 同一分支的实际物理距离等于该分支下所有片段长度求和。
5. 实际物理距离用于运行、倒车和干涉判断；股道可停车有效长度仍以股道主数据表和 `trackDistance` 为准，二者不能混用。
6. 警冲标和岔区可以计入运行物理距离，但不能计入停车占用长度。

## 6. 车辆分类与业务属性

### 6.1 车型分类

开发阶段至少区分：

1. 罐车
2. 棚车
3. 敞车
4. 平车

补充口径：

1. `重车` 不是车型，而是车辆当前装载状态或运营属性。
2. 同一辆车可以同时是 `敞车` 和 `重车`；现场所说“营运重车”不改变其车型仍为敞车这一事实。

### 6.2 修程分类

| 修程类别 | 业务含义 | 典型周期口径 |
| --- | --- | --- |
| 段修 | 常规周期性检修 | 约 1 至 1.5 年 |
| 厂修 | 周期更长、修程更深 | 约 2 至 3 年 |
| 临修 | 故障处置 | 不按固定周期，按故障触发，一般不进大库 |

### 6.3 关键业务属性

1. 称重车：本轮流程中需要经过机库称重工位。
2. 关门车：定义见 3.1；业务上主要影响进段分流和修竣集结。
3. 关门车的详细顺位限制统一见《福州东调车业务说明》。

## 7. 标准业务流程

### 7.1 主流程

`扣车计划 -> 存5接车 -> 信息判定 -> 形成本轮调车目标 -> 执行本轮计划 -> 到达目标工序或阶段性停放 -> 工序完成判定 -> 形成下一轮调车目标 -> 多轮循环 -> 全部工序完成 -> 修竣集结 -> 存4北交站`

### 7.2 进段与信息判定

车辆进入存5后，至少要得到以下业务结论：

1. 车号
2. 修程类别
3. 是否重车
4. 是否称重车
5. 是否关门车
6. 本轮明确调车目标
7. 当前工序完成后是否还存在下一轮工序目标

补充说明：

1. 车辆通常依据扣车计划由车站送入段内 `存5北`、`存5南`。
2. 车辆进段后会先经历预检或信息判定，再形成当轮调车目标。
3. 第一目标由上游业务明确给定，不由本文自动推导。

### 7.3 典型工序流转背景

下表用于说明福州东车辆段常见业务流，是业务背景，不是算法自动推导规则：

| 车辆类别 | 常见首去向 | 常见后续流向 | 常见终点 |
| --- | --- | --- | --- |
| 非罐车功能检修 | 预修能力组 | 功能作业区或大库 | 存4北 |
| 需要调棚作业车辆 | 调棚作业区 | 预修或功能作业区 | 存4北 |
| 非临修罐车 | 洗南 | 功能作业区或大库 | 存4北 |
| 临修罐车 | 洗南 | 一般存车区或存4北 | 存4北 |
| 营运重车/装卸车 | 轮 | 一般存车区或存4北 | 存4北 |

补充说明：

1. `预修`、`调棚`、`抛`、`油` 等工序完成后，车辆通常继续转往大库。
2. 非临修罐车洗罐完成后，通常继续转往大库；临修罐车洗罐完成后可直接转往存4北相关出段流程。
3. 轮线既可接收存5来的营运重车，也可承接大库修竣后的装轮、托运等后续运营作业。

### 7.4 修竣集结与出段

1. 全部工序完成后，可出段车辆最终统一向存4北集结。
2. 存4南只作为过渡临停，不作为正式出段终点。
3. 关门车在修竣集结及其他相关钩计划中仍需满足专门顺位限制；进入存4北与非存4北两类场景的具体判定，统一见《福州东调车业务说明》。
