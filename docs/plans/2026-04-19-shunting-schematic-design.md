# Shunting Schematic Replay Design

**Date:** 2026-04-19

## Goal

把当前 `app.py` 中偏工程底图式的线路回放，改成“地铁式方位示意图 + 右侧钩信息栏 + 底部细节区”的结构。主视图优先服务单钩回放：用户应当一眼看懂“从哪里来，经哪条主干，拐去哪里”，而不是同时消化真实几何、全量股道状态卡片和距离目录。

## Why Change

当前实现存在三个结构问题：

- 主回放图仍带有较强的真实底图痕迹，视觉上更接近工程底图高亮，不像稳定的示意图
- `逐钩回放` 中同时堆叠了拓扑图、状态卡片、股道列表，主次不清
- `segmented_routes_svg.py` 里的距离实验图把“全局线网”和“距离目录”压在同一张图里，不适合直接承担主页面角色

因此这轮改动不继续打磨旧总图，而是把信息拆层：

- 主图回答：怎么走
- 右栏回答：这一钩在干什么
- 底栏回答：细节证据是什么

## Information Architecture

### 1. Replay Tab

主页面保留为回放页，但内部结构调整为：

- 顶部：现有场景/plan/solver 控件
- 中部：单钩回放主区域
- 左侧 `7`：地铁式方位示意图
- 右侧 `3`：当前钩摘要卡
- 底部：4 个细节 tab

细节 tab：

- `当前状态`：只显示当前有占用或本步变化的股道
- `车辆明细`：当前钩涉及车辆、路径、目标
- `校验结果`：PASS/FAIL 与关键错误
- `距离构成`：当前钩路径的股道级距离拆解

### 2. 线路总览 Tab

单独提供一张静态地铁式总览图，用于理解全场结构，不承载实时状态。

特点：

- 用固定示意布局，不随步骤变化
- 保留业务区域底色
- 常显主干与关键终点标签
- 不显示车辆状态卡片

### 3. 距离构成 Tab

把原本塞在主图旁边的距离目录拆出来，独立展示：

- 每条聚合物理分支一行
- 展示分支名、左右端、段列表、总距离
- 保留 `segmented_physical_routes.json` 的价值，但不再挤占主回放空间

## Visual Grammar

### Main Schematic

主图采用稳定的方位示意图，不再直接使用背景图片。画法规则：

- 主干固定成一条稳定中轴
- 仅使用 45/90 度折线
- 所有支线以“挂载”方式连接到主干
- 区域只用很淡的背景块表达，不参与状态编码

主干顺序：

`联6 -> 渡2 -> 临1 -> 临2 -> 渡5 -> 渡6 -> 渡7 -> 预修 -> 渡9 -> 渡10 -> 联7 -> 渡12 -> 渡13`

区域挂载：

- 上方：存车区（存5/存4/存3/存2/存1）
- 中右：调车/机库区（渡4/调北/调棚/机库）
- 右下：机棚/油洗区（机北/机棚/临3/油/洗北/洗南）
- 左下：检修区（抛/轮/修1-4）

### Highlight Rules

- 底图骨架：细浅灰线，只表达结构
- 当前路径：蓝绿色粗线，为唯一主色
- 起点/终点：使用明显端点标记
- 机车位置：单独橙色标记
- 变化股道：琥珀色描边，不抢主路径颜色
- 非当前路径但有占用：仅用小徽标或轻标签提示
- 校验失败：仅在右栏和相关股道附近使用红色

### Labels

默认常显：

- 主干关键股道
- 关键终点：`机库`、`调棚`、`油`、`洗南`、`修1-4`
- 区域名

其它股道按需提升：

- 当前钩路径上的股道升为主字号
- 非当前路径侧线保持次级字号

## Layout Source of Truth

不再让主回放示意图依赖自动布局。新增一份固定布局数据，作为示意图单一真源：

- `data/master/schematic_layout.json`

内容包含：

- 画布尺寸
- 分区背景块
- 每条股道的折线坐标
- 标签锚点
- 哪些股道默认常显

自动布局实验仍保留在 `segmented_routes_svg.py`，但只作为距离构成视图的来源，不再承担主回放图职责。

## Non-Goals

- 不改求解器、回放、verifier 行为
- 不把示意图做成真实 GIS/园区底图
- 不在本轮接入复杂 hover/click JS 交互
- 不重写 `segmented_routes_svg.py` 的实验布局算法
