# 调车计划生成 WebAPI 接口文档


---

## 1. 生成调车计划

* **功能描述**：根据当前站内车辆的初始位置、目标线路及各物理股道的配置，返回调车机车的详细作业指导步骤（勾计划）及计算后的实际对位结果。
* **请求路径**：`/api/plan/generate`
* **请求方式**：`POST`
* **Content-Type**：`application/json`

### 1.1 请求参数 (Request Body)

请求体需传递一个 JSON 对象，包含2个核心列表：`StartStatus`（起点状态）和 `TerminalLines`（库内股道配置）。

#### 根对象 (`PlanCreatePostData`)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `StartStatus` | `List<StartCarDto>` | 是 | 车辆的初始状态列表。 |
| `TerminalLines` | `List<TerminalLineDto>` | 是 | 终点规划器可用的目标股道配置信息。 |
| `locoTrackName` | `String` | 是 | 调车机当前的位置 |

#### `StartCarDto` (初始状态)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `Line` | string | 是 | 起点股道名称，例如：`"修1线大库外"`。 |
| `Position` | int | 是 | 起点连续台位号，非负整数。 |
| `RepairProcess` | string | 是 | 修程 |
| `Type` | string | 是 | 车型信息。 |
| `No` | string | 是 | 车号。 |
| `Length` | decimal | 是 | 车辆长度，单位m |
| `IsHeavy` | bool | 否 | 是否为重车，默认 `false`。 |
| `IsWeigh` | bool | 否 | 是否需要称重，默认 `false`。 |
| `IsClosedDoor` | bool | 否 | 是否是关门车，默认 `false`。 |
| `TargetLines` | list | 是 | 目标股道名称。如有多个候选线（面到线退化），可用逗号分隔，如 `["调梁棚","调梁棚"]`。 |
| `ForceTargetPosition` | list | 否 | 新版强制对位文本，支持逗号隔开的多个绝对台位，例如 `"["1","2"]`,大库区域也使用每个股道的台位，而不是带股道名称。 |

#### `TerminalLineDto` (库内股道配置)

| 字段名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `Line` | string | 是 | 目标股道 First Name。 |
| `IsSpotting` | bool | 否 | 是否开启迎检模式。任意一条线开启后，全局库内台位容量扩展为 7（否则默认为 5）。 |

---

### 1.2 响应参数 (Response Body)

返回标准响应结构 `PlanGenerateResponseDto`，核心数据在 `Data` 字段中。

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `Success` | bool | 业务处理是否成功。 |
| `Message` | string | 提示信息或异常信息。 |
| `StatusCode` | int | HTTP 状态码。 |
| `Data` | `PlanGenerateResultDto` | 处理成功时返回的数据载荷。 |

#### `PlanGenerateResultDto` (动作信息，提取关键字段)

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| `Index` | int | 动作序号。 |
| `LineDisplayName` | string | 展示用完整股道名（如 `"修1库内"`, `"存5线北"`）。 |
| `Action` | string | 操作动作：`Get` (取车), `Put` (放车), `Weigh` (称重)。 |
| `MoveCars` | `List<Car>` | 参与本次移动的车辆列表。 |
| `TrainCars` | `List<Car>` | 操作后机车上承载的车辆列表。 |
| `PassbyPath` | `List<str>` | 从上一勾机车位置到当前勾经过的线路和道岔的列表（如：存1线到洗罐线北：存1线，L5，L6，Z1，L8，L9，洗罐线北）（如：修4库外到修3库外：修4库外，L18，修3库外） |
