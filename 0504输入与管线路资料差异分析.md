# 0504 输入规则与管线路资料差异分析

## 1. 对比范围

本次对比的新资料：

1. `调车钩计划算法需求说明书_输入0504.xlsx`
2. `附件3：福州东车辆段段内管线路简表（新）0504.docx`

对比基准：

1. `福州东车辆段业务说明.md`
2. `福州东调车业务说明.md`

结论口径：

1. Excel 0504 主要更新调车规则、缓存规则、钩数优化规则和点到线/点到区域落位规则。
2. Word 0504 主要更新段内管线路规范简称、有效长度、停留属性和业务备注。
3. 现有两份 Markdown 已经吸收了一部分旧规则，但和 0504 新资料之间存在若干明确冲突，尤其是 `L1` 道岔长度限制、大库自动台位分配、股道规范简称、若干线路有效长度。

### 3.2 明确需要同步的新增或变更规则

#### 3.2.1 `L1` 道岔长度限制从 `190m` 改为 `68m`

0504 Excel 中 R007 出现两行：

| 状态 | 规则 | 规则描述 |
| --- | --- | --- |
| 已废弃 | 长度限制：`190` 米 | 经过 `L1` 道岔时，机后连挂车辆长度 `<= 190m` |
| 已确认 | 长度限制：`68` 米 | 经过 `L1` 道岔时，机后连挂车辆长度 `<= 68m` |

现有 `福州东调车业务说明.md` 第 5.2 节仍写“经过 `L1` 道岔时，机后车辆总长度不得超过 `190` 米”。这是明确冲突，不是补充说明。

建议：

1. 调车业务说明中的硬约束应改为 `68m`。
2. 代码中所有 `L1` 经过约束、测试数据和异常说明也应同步。
3. `福州东车辆段业务说明.md` 中 `大门L1=190` 属于运行物理距离分支，不应直接等同于 `L1` 道岔机后连挂长度限制；两者需分开建模。

#### 3.2.2 新增“单批称重车最多 2 辆”

0504 R005 明确：

| 规则 | 规则描述 |
| --- | --- |
| 单批称重车 `<= 2` 辆 | 单批次调车最多 `2` 个称重车 |

现有文档只约束“单钩称重最多处理 `1` 辆称重车”，没有批次级上限。

建议补充：

1. 计划输入级别统计 `needWeigh=true` 车辆数量，不得超过 `2`。
2. 单钩仍保留最多 `1` 辆称重车，两条规则不互相替代。

#### 3.2.3 新增缓存区域规则

0504 R013-R016 明确：

1. `Z1、Z2、Z3` 不设为缓冲区。
2. 附件3中的临时停车区域只能用于缓存，不能作为最终终点。
3. 附件3中的存车线路也能作为缓存。
4. 附件3中的存车线路为大库时，只针对大库之间的车辆缓存。

现有文档已经有“临停线不能作为正式终点”，但未完整覆盖 `Z1/Z2/Z3` 不作为缓冲区，以及“大库存车线路缓存只服务大库之间车辆”的边界。

建议：

1. 将 `Z1/Z2/Z3` 明确为路径/岔区节点，不允许缓存、临停和终点。
2. 缓存候选线分层：
   - 临时停车区域：只允许过程缓存。
   - 普通存车线路：允许过程缓存，也允许作为合法终点。
   - 大库相关存车线路：只允许在大库相关搬移、库内外周转中作为缓存，不能泛化为全局缓存。

#### 3.2.4 新增钩数优化约束：取车钩合并、南北分钩、库内外分钩

0504 R017-R019 明确：

| 规则 | 规则描述 | 优先级 |
| --- | --- | --- |
| 取车钩合并 | 充分利用机后所挂车辆，对部分取车钩进行合并，减少钩数 | 中 |
| 南/北分钩 | 当同一条线路涉及南北挂车和摘车时需分两钩 | 高 |
| 库内/库外分钩 | 当同一条线路涉及库内外挂车和摘车时需分两钩 | 高 |

现有 `福州东调车业务说明.md` 只有“总勾数最少”的优化目标，没有把这些作为生成钩计划时的结构性规则。

建议：

1. “取车钩合并”作为通用策略：只要机后车辆集合、牵引限制、路径方向、目标顺序和中间放置都合法，就应合并可合并的取车动作。
2. “南/北分钩”作为硬约束：同一线路上若需要从南北两个方向分别处理车列，不应在同一钩中混做。
3. “库内/库外分钩”作为硬约束：大库库内和库外虽然物理相邻，但取放边界不同，不能为了减少钩数强行合并。

### 3.3 点到线落位规则的新细节

0504 R021-R037 对各线路落位规则作了逐线确认。

已基本一致的部分：

1. `存1`、`存2`、`存3`、`存5南`、`机库`、`预修`、`机棚`、`调北`、`洗北`、`轮`：车辆在股道内随机分配。

3. `洗南`：对位车辆进入作业区，功能限制 `3` 个台位。
4. `油`、`抛`：对位车辆进入库内/最南端作业区，功能限制 `2` 个台位。

需要同步或澄清的部分：

### 3.4 大库库内自动分配规则存在重大冲突

0504 R039 对大库库内区域放置规则如下：

1. 指定台位车辆按预先指定台位分配。
2. 剩余空余台位随机分配，优先级：
   - 车长 `>= 17.6m` 的厂修车只能分配至 `304、305、404、405`。
   - 车长 `< 17.6m` 的厂修车优先分配至 `104、105、204、205`，如 `304、305、404、405` 未占用，也可分配。
   - 车长 `>= 17.6m` 的非厂修车只能分配至 `301~403`，且不含 `4、5` 台位。
   - 车长 `< 17.6m` 的非厂修车分配至 `101~203`，且不含 `4、5` 台位；如 `301~403` 中不含 `4、5` 台位未占用，也可分配。
3. 各库 `06、07` 台位为虚拟台位，仅手动指定，空置时不参与自动随机分配。

现有 `福州东调车业务说明.md` 第 8 章口径：

1. 不再把段修、厂修写成两套固定物理区域。
2. `NORMAL` 每库使用 `01-05`。
3. `INSPECTION` 每库可使用 `01-07`。
4. 车长 `>= 17.6m` 使用 `301-305、401-405`；车长 `< 17.6m` 优先 `101-105、201-205`，不足再到 `301-305、401-405`。

冲突点：

| 主题 | 0504 R039 | 现有 Markdown |
| --- | --- | --- |
| 是否区分厂修/非厂修 | 明确区分 | 明确不再固定区分段修/厂修物理区域 |
| 长车厂修可用台位 | 只允许 `304、305、404、405` | 允许 `301-305、401-405` |
| 长车非厂修可用台位 | `301~403`，不含 `4/5` 台位，即偏向 `301-303、401-403` | 允许 `301-305、401-405` |
| 短车厂修优先台位 | `104、105、204、205` | `101-105、201-205` |
| `06/07` 台位 | 仅手动指定，自动随机不使用 | `INSPECTION` 模式可自动扩展使用 |

建议：

1. 该项不能直接默默合并，必须作为业务口径冲突处理。
2. 如果 0504 R039 是最新确认规则，应替换现有第 8 章的大库自动分配规则，并把 `06/07` 从“迎检可自动使用”改为“仅手动指定”。
3. 如果现有 Markdown 的 `INSPECTION` 口径来自更晚人工确认，则需要在差异文档中标注 0504 Excel 与人工确认冲突，并保留现有规则。

### 4.5 停放/作业属性变化

0504 附件3最后注意事项明确：

1. 有效长度是指用于临停和终点停车的长度，备注为不可停留车辆的不参与。
2. 系统及算法采用规范简称。
3. 车辆检修作业区域、存车线路即可临停也可终点。

这对现有文档有三处影响：

1. `洗北`：0504 备注为“车辆检修作业区域”，现有文档把它作为“存车线/洗南前后缓存”。需要确认 `洗北` 是否仍可作为终点，还是应归入检修作业区域。
2. `机库`：0504 备注为“存车线路，不执行车辆检修作业”，现有文档把它列为专用线并允许称重和最终目的地。二者不必冲突，但需要补一句：0504 附件3只定义其停车属性，称重工位属于额外业务能力。
3. `存4北`：0504 备注为普通“存车线路，不执行车辆检修作业”，现有文档赋予“出段集结线”业务角色。二者可兼容，但主数据类型和业务角色应分开。
